# Database Fundamentals

## What is a Database?

A database is an organized collection of data that allows information to be stored, managed, updated, and retrieved efficiently.

Example:

- Student Records
- Bank Accounts
- Hospital Data
- E-commerce Products

---

# Database Flow

User

↓

Application

↓

Database Management System (DBMS)

↓

Database

↓

Result returned to User

Explanation:

1. User requests information.
2. Application sends the request.
3. DBMS processes the request.
4. Database stores or retrieves data.
5. Results are sent back to the user.

# SQL Database

SQL databases store data in tables.

Features:

- Rows and Columns
- Fixed Schema
- Structured Data
- Uses SQL language

Examples:

- MySQL
- PostgreSQL
- SQL Server

---

# MongoDB

MongoDB is a NoSQL database.

Features:

- Stores data as JSON-like Documents
- Flexible Schema
- Easy to scale
- High performance

Example Document

{
  "name": "Ali",
  "age": 22,
  "city": "Lahore"
}

---

# MongoDB Atlas

MongoDB Atlas is MongoDB's cloud database service.

Learned:

- Creating an account
- Creating a cluster
- Connecting to a database

Benefits:

- Cloud hosted
- Secure
- Easy deployment
- Accessible from anywhere

---

# MongoDB Compass

MongoDB Compass is a graphical interface (GUI).

Purpose:

- View databases
- Create collections
- Insert documents
- Edit data
- Run queries visually

No command line required.

---

# MongoDB Shell

MongoDB Shell (mongosh) is the command-line interface.

Purpose:

- Connect to databases
- Execute commands
- Insert documents
- Update records
- Delete records
- Run queries

Useful for developers and administrators.
